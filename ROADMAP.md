## Phase 2 — 线激光三角测量（扫地机 3D 避障核心）

**目标**：理解线激光传感器的工作原理，用相机 + 线激光模组实现 3D 轮廓扫描。

### 扫地机中的应用

Roborock ReactiveAI / Dreame 等避障系统：线激光发射器在前方投射一条横线，红外相机从侧面观察激光线的形变，通过三角测量计算出每个位置的 3D 高度。这也是工业上"激光轮廓仪"的原理。

```
                激光发射器
                   |
                   | 投射一条横线
                   v
    ════════════════════════════  ← 激光线投射到地面/障碍物
              /\
             /  \  相机从上方看到激光线弯折
            /    \
          相机 (侧上方)
```

### 知识点

#### 激光三角测量原理
- **基本几何**：激光投射点 → 物体表面反射 → 相机成像点，三者构成三角形
- **核心公式推导**：
  ```
  Z = (B * f) / (x + B * f / Z₀)
  ```
  其中 B = 基线距离（激光-相机间距），f = 焦距，x = 像点偏移
- **光平面标定**：确定激光平面在相机坐标系中的位置（比相机标定多一步）
- **线激光 vs 点激光 vs 结构光**：速度、精度、成本对比

#### 激光线中心提取
- 灰度重心法（最常用，简单）
- 极值法
- 高斯拟合法（亚像素精度）
- Steger 算法（亚像素精度，工业标准）
- 骨架细化（Zhang-Suen）

#### 3D 扫描流程
- 激光线投射 → 相机采图 → 提取激光线中心 → 三角计算 → 得到一行 3D 点
- 物体移动（或传感器移动）→ 逐行拼接 → 完整 3D 轮廓
- 点云生成与可视化

### 所需硬件

| 物料 | 预算 | 说明 |
|------|------|------|
| 650nm 红色线激光模组 | ~30-50 元 | 淘宝搜"线激光模组"，5mW 即可，选可调焦距的 |
| USB 摄像头（已有）| 0 | 需要能拍红外更好（去掉 IR 滤光片），普通摄像头也能用 |
| 滑轨/直线导轨（可选）| ~50 元 | 手动推也行，先不买 |

2. **光平面标定**：用棋盘格 + 激光线的几张照片，标定光平面参数
3. **3D 轮廓扫描**：把激光线照在一个台阶状物体上，算出 3D 高度

## Phase 3 — LDS 激光雷达与 2D SLAM（扫地机建图核心）

**目标**：理解 LDS（Laser Distance Sensor）原理，读取激光雷达数据，跑通 2D 栅格建图。

### 扫地机中的应用

扫地机顶部旋转的"小塔"就是 LDS。它一边旋转一边发射激光，每次测量一个角度上的距离，转一圈得到 360° 的 2D 点云。通过 Hector SLAM / Cartographer 算法将连续的点云拼接成房间地图。

### 知识点

#### 三角法激光雷达
- **原理**：激光发射 → 物体反射 → 镜头汇聚到线阵 CMOS/CCD → 光斑位置对应距离
  ```
  距离越近 → 反射光斑在传感器上的偏移越大
  ```
- **与 ToF 雷达对比**：三角法便宜但精度随距离下降（适合 0.1-10m），ToF 精度稳定但贵
- **YDLIDAR / RPLIDAR 拆解**：结构组成（激光管、镜头、线阵传感器、旋转机构、磁耦合供电）

#### 数据读取
- 串口通信（UART @ 115200 或更高）
- 数据格式：角度、距离(mm)、信号强度、转速(RPM)
- Python SDK / ROS 驱动

#### 2D SLAM 基础
- **占据栅格地图（Occupancy Grid Map）**：概率模型，0=空闲，100=占据，-1=未知
- **Hector SLAM**：基于 scan-to-map 匹配，不需要里程计
- **Cartographer**：Google 的方案，效果更好但计算量大
- **坐标变换**：激光雷达坐标系 → 机器人坐标系 → 世界坐标系

### 所需硬件

| 物料 | 预算 | 说明 |
|------|------|------|
| YDLIDAR X4 | ~300 元 | 入门级三角法雷达，10m 范围，5000 points/s |
| 或 RPLIDAR A1 | ~400 元 | 思岚的入门型号，SDK 更成熟 |
| USB-TTL 模块 | ~10 元 | 如果雷达不是 USB 接口 |

### 动手练习

1. **读取雷达数据**：用 Python 串口读取一帧 360° 点云，可视化
2. **点云 → 极坐标显示**：用 Matplotlib 实时绘制周围环境
3. **栅格建图**：用 Hector SLAM（Python 实现或 ROS 版）跑通建图
4. **手持雷达建图**：拿着雷达在房间里走一圈，生成地图

### 验证标准

- [ ] 能解释三角法激光雷达的距离计算公式
- [ ] 能从串口读取激光雷达数据并实时可视化

## Phase 4 — 红外传感器与 ToF（近距离感知）

**目标**：理解扫地机上各种红外传感器的工作原理，驱动 VL53L0X ToF 传感器。

### 扫地机中的应用

扫地机底盘边缘密布多种短距传感器：
- **悬崖传感器**：3-4 个向下的红外对管，检测台阶/楼梯
- **墙距传感器**：侧面的红外测距，贴墙清扫
- **碰撞传感器**：红外对管 + 挡板（物理缓冲 + 红外触发）
- **ToF 传感器**（部分型号）：前方近距离精确测距
- **沿边红外**：斜向下红外，判断是否贴近墙边

### 知识点

#### 红外对管
- 红外 LED 发射 + 光电三极管接收
- 反射式：检测反射光强度（悬崖传感器原理）
- 对射式：检测光是否被遮挡（编码器、碰撞检测）
- 三角测距红外（Sharp GP2Y0A 系列）：PSD 位置检测器
- 局限与对比：颜色/材质反射率影响、环境光干扰

#### ToF 传感器（VL53L0X / VL53L1X）
- **飞行时间原理**：发射红外脉冲 → 接收反射 → 计算时间差 → 距离
- SPAD（单光子雪崩二极管）阵列
- Histogram 算法：多光子统计提高精度
- VL53L0X vs VL53L1X：1m vs 4m 范围，单区 vs 多区（ROI）
- 与三角法对比：ToF 不受物体颜色影响

#### 传感器选型对比
| 类型 | 范围 | 精度 | 受颜色影响 | 成本 |
|------|------|------|-----------|------|
| 红外对管（反射式）| 1-15cm | 低 | 严重 | ~1 元 |
| 红外三角（Sharp）| 10-150cm | 中 | 中等 | ~30 元 |
| ToF（VL53L0X）| 3-200cm | 高(±3%) | 几乎无 | ~15 元 |
| 超声波 | 2-400cm | 低 | 无 | ~5 元 |

### 所需硬件

| 物料 | 预算 | 说明 |
|------|------|------|
| VL53L0X / VL53L1X 模块 | ~15 元 | I2C 接口，Arduino 或树莓派都能驱动 |
| 红外对管模块 | ~2 元/对 | 淘宝搜"红外避障模块" |
| Arduino Nano 或直接 USB-I2C | ~30 元 | 用树莓派或电脑的 I2C 也行 |

### 动手练习

1. **VL53L0X 测距**：用 Python (smbus2) 通过 I2C 读取距离，对比实际值
2. **红外对管实验**：测试不同颜色/材质的反射率差异
3. **悬崖检测**：把红外对管朝下，检测桌面边缘
4. **多传感器对比采集**：同时采集 3-4 种传感器的数据，对比响应

## Phase 5 — 双目视觉与深度相机

**目标**：理解双目立体匹配原理，对比双目、结构光、ToF 三种深度方案。

### 扫地机中的应用

- 部分高端扫地机（iRobot Roomba j7+、部分 Roborock 型号）用 RGB 双目做前方避障
- 结构光（红外散斑投射 + 红外相机）用于夜间或弱光环境
- 典型深度相机：Intel RealSense D435（双目 + 结构光辅助）、奥比中光 Astra

### 知识点

#### 双目立体视觉
- **对极几何**：本质矩阵 E、基础矩阵 F
- **立体校正（Bouguet）**：让左右图极线水平对齐
- **立体匹配**：BM（Block Matching）、SGBM（Semi-Global Block Matching）
- **视差 → 深度**：
  ```
  Z = B * f / d
  B=基线, f=焦距, d=视差(像素)
  ```
- **精度分析**：
  ```
  ΔZ = (Z² / (B*f)) * Δd
  距离越远 → 精度越差（平方关系）
  基线越长 → 精度越好，但盲区越大
  ```

#### 结构光
- 散斑投射（Kinect v1 / RealSense）
- 编码结构光：格雷码、相移法
- 与线激光对比：面阵结构光更快，但精度通常不如线激光

#### 三种深度方案对比
| 方案 | 精度 | 速度 | 弱光 | 室外阳光 | 成本 |
|------|------|------|------|----------|------|
| 双目 | 中 | 快 | 需补光 | 纹理够可以用 | 低 |
| 结构光 | 高 | 快 | 好 | 差（阳光淹没）| 中 |
| ToF | 中 | 极快 | 好 | 一般 | 中高 |
| 线激光扫描 | 高 | 慢（需运动）| 好 | 好 | 低 |

### 动手练习

1. **双目标定 + 立体匹配**：用两个 USB 摄像头拍棋盘格 → 立体标定 → 生成视差图
2. **深度精度验证**：在 1/2/3/5m 处放物体，对比测出的深度和实际距离
3. **视差调参实验**：改 SGBM 的 numDisparities 和 blockSize，看对精度/噪声的影响
4. **点云生成**：从深度图 + 相机内参 → 生成 XYZ 点云 → Open3D 可视化

- [ ] 能独立完成双目立体标定 + 立体校正
- [ ] 能解释双目在什么场景会失效（白墙、重复纹理、弱光）

## Phase 6 — AI 避障识别

**目标**：用目标检测模型识别扫地机特有的障碍物（线缆、宠物粪便、鞋子、袜子等）。

### 扫地机中的应用

- Roborock ReactiveAI 2.0：RGB 相机 + 神经网络识别 62 种物体
- 关键障碍物类型：线缆（最容易缠住）、宠物粪便（灾难性）、鞋子/袜子/玩具
- 模型需要在嵌入式芯片上实时运行（~30ms 以内）

### 知识点

#### 模型选型
- YOLO nano/small（平衡精度和速度）
- MobileNet-SSD（更轻量）
- NanoDet（专门为移动端设计的轻量检测器）
- 模型量化：INT8/FP16 推理

#### 嵌入式部署
- ONNX → ONNX Runtime（已完成基础实验）
- TensorRT（NVIDIA Jetson）
- OpenVINO（Intel）
- 树莓派 + Hailo-8L / Google Coral TPU

#### 扫地机特殊问题
- **距地面 5-15cm 的低视角**：物体外观变形大，需要低视角数据
- **线缆检测**：细长物体，传统检测框效果差，需要特殊的线缆分割模型
- **地面反光**：地板/瓷砖反光造成误检
- **实时 vs 精度**：扫地机速度慢（~0.3m/s），允许 200ms 的推理延迟

### 动手练习

1. **收集扫地机视角数据**：用手机在家贴地面拍 100 张（线缆、鞋子、袜子、充电线）
2. **标注训练**：用 LabelIMG 标注 → 在预训练 YOLO 上微调
3. **模型量化对比**：导出 ONNX INT8，对比精度损失和速度提升
4. **异常案例收集**：地板反光误检的照片，思考怎么处理

- [ ] 能独立完成数据收集 → 标注 → 训练 → 评估的闭环
- [ ] 训练的模型对线缆/鞋子的检测 mAP > 0.7
- [ ] 理解模型量化（INT8）对推理速度和精度的 tradeoff
- [ ] 能说出扫地机低视角和通用 COCO 数据集（平视视角）的关键差异

## Phase 7 — 搭建迷你扫地机感知系统

**目标**：把 Phase 2-6 学到的传感器整合起来，搭建一套完整的感知系统原型。

│                                                  │

│                      │                          │

### 知识点

#### 系统集成
- 多线程采集：每个传感器一个采集线程
- 数据格式统一：所有障碍物统一为 (x, y, z, class, confidence) 格式
